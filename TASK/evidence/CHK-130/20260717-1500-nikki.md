## 3. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Order result (api) | `GET /v1/orders/{orderId}` | Schema + mapper; session cookie SSR; `privacy: private` → no-store |
| Guest capability | optional `X-Order-Capability` | Memory only; never query/path/storage/log |
| Path `[status]` | presentational | Ignored as authority; mismatch → redirect to backend-derived path |
| Foreign/missing | 404 | `null` / `notFound()` — no existence enumeration |
| Success chrome | existing markup | Gross/title/order id from BE; download button has no action (CHK-140) |
| Mock | adapter fixture | `paymentStatus: PAID` fixture; still not URL-driven |

### Authority

- Only backend `paymentStatus` / `orderStatus` select success | pending | failed.
- Modifying URL `/success` vs `/failed` cannot change display without BE state (redirect or ignore).
- Browser query `total`/`tip` not used on this page.
- Delivery secrets never mapped on base result.

### Capability policy

- Authenticated owner: session cookie forward (INT-110/120).
- Guest: purpose-bound token only via header after memory/fragment exchange (no query).
- ID alone insufficient at policy level; BE may still harden further.
