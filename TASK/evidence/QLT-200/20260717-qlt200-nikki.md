## 2. Coverage policy (honest denominator)

**Before:** `include` was only `shared/api/http-client.ts` + `shared/ui/pagination.ts` with 85% thresholds → false confidence.

**After:** include live adapters/schemas/mappers/policies/session/source:

- `shared/api/**` (except `generated/**`)
- `shared/data`, `shared/query`, `shared/auth`, `shared/seller`, `shared/finance`, `shared/notifications`
- `features/**/api.ts`, `mappers.ts`, `*policy*`, `server.ts`

**Exclude (written):** generated OpenAPI TS (drift via `api:check`), mock/demo fixtures, presentation TSX/hooks, type barrels.

**Thresholds** (against expanded set): lines/statements 55%, functions 50%, branches 45%. Measured ~71% statements / ~74% lines / ~59% branches on 2026-07-17 full unit+contract run — gates pass without shrinking denominator.

## 3. Contract pattern

```text
OpenAPI schema / provider DTO fixture
  → FE Zod runtime schema (safeParse)
  → pure mapper → existing view model
  → fail closed INVALID_API_CONTRACT
```

- **Provider (Go):** presenter envelope + CatalogProduct required fields vs OpenAPI components.schemas.CatalogProduct.
- **Consumer (FE):** foundation catalog happy/empty/malformed/float-money/unknown-type/problem/request-body keys.
- **Parity:** BE `featured-products.provider.json` loads through FE schema+mapper.
