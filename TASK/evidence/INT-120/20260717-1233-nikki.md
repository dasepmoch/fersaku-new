# INT-120 — Session provider, bootstrap, logout, route guards

**Status:** `[x]` completed  
**Actor:** @nikki/opencode  
**When:** 2026-07-17 12:33 WIB  
**Depends on:** INT-025, INT-100, INT-110, INT-130 (all `[x]`)

## 1. Objective delivered

Single source-neutral session authority for buyer/seller/admin:

- Bootstrap via `GET /v1/auth/session` (API) or mock claims only when `auth` domain source is `mock`
- Session model without raw session cookie / CSRF (CSRF stays INT-130 memory)
- Private route guards wait for bootstrap; safe `returnTo`; server-side logout + private cache clear
- No UI redesign — existing login shells and profile markup retained

### Wiring (no redesign)

- Root: `SessionProvider` inside `AppQueryProvider` (`app/layout.tsx`)
- Buyer: `app/account/layout.tsx` → `BuyerAccountShell`
- Seller: `app/dashboard/(workspace)/layout.tsx` → `PrivateSurfaceShell surface="seller"`
- Admin: `app/admin/(console)/layout.tsx` → `PrivateSurfaceShell surface="admin"`
- `AdminPermissionBoundary` uses session claims (API mode never hardcodes mock admin)
- `ProfileMenu` logout → `logoutSession` (backend + CSRF clear + private cache + redirect)

### CSRF integration (INT-130)

- Bootstrap / login path: `setCsrfToken` from session/login body
- `wireHttpClientCsrfHooks()` on provider mount
- Logout / expiry: `clearCsrfToken()`

## 4. Route behavior

| Surface | Missing session | Wrong surface |
| --- | --- | --- |
| Buyer | `/account/login?returnTo=...` | `/account/login` |
| Seller | `/login?returnTo=...` | `/login` |
| Admin | `/admin/login?returnTo=...` | `/admin/login` only |

- Private paths render children only after bootstrap `ready` + guard `allow`
- Auth entry redirects authenticated same-surface users to surface home
- `returnTo`: relative path only; scoped to surface prefix; rejects `//`, absolute URLs, schemes
