## 3. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| List (api) | `GET /v1/buyer/sessions` | `{ sessions: SessionView[] }`; `current` = BE session-id equality |
| Revoke one | `POST /v1/buyer/sessions/{sessionId}/revoke` | Own only; foreign → 404 |
| Revoke others | `POST /v1/buyer/sessions/revoke-others` | Single bulk call; keep current |
| Revoke all | `POST /v1/buyer/sessions/revoke-all` | BE clears cookie; FE clears private cache + redirect |
| Revoke current | same single revoke | `revokedCurrent` via claims.sessionId; then INT-120 local clear + `/account/login` |
| Mock buyer | fixtures | demo sessions; no network |
| Privacy | location/ip | masked `"—"` (BE buyer list has no raw IP); device text sanitized |

### Password / MFA disposition

- Buyer security snapshot is **passwordless** (magic-link copy only). No password/MFA enroll controls on this screen.
- AUT-120 remains owner of shared password/MFA ceremony; not invented here.

### Security rules

- Session-bound list/revoke of **own** sessions only.
- No optimistic session removal.
- Bulk “Keluar dari perangkat lain” never loops single-revoke.
- Logout/revoke-all/current clears cookies (server Set-Cookie clear) + private RQ cache + session store.
