## 3. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Create (api) | `POST /v1/buyer/reviews` | Body: `orderItemId`, `rating`, optional `title`/`body`/`productId`/`storeId` — **no status** |
| Patch (api) | `PATCH /v1/buyer/reviews/{reviewId}` | `expectedVersion` + content; version conflict 409 rethrows (form draft kept) |
| Mock buyer | adapter fixtures | Deterministic review; no network |
| Success | invalidate | purchase detail/list, buyer review key, public product reviews + summary |
| Eligibility/ownership | server | 403 not eligible / 404 non-owner — no fake success |

### Version-update (“Gunakan versi baru”) disposition

- **DISABLED** when `buyer=api`: control remains visible only if mock fixture sets `updateAvailable` + `sellerUpdatesEnabled`; API mappers always set `sellerUpdatesEnabled: false`.
- Never satisfied by `setUpdated(true)` as a real entitlement command in api mode.
- Requires future BE freeze (entitlement revision vs latest delivery read) before IMPLEMENT.

### Resend / download / invoice

- Already wired under CHK-140 / CHK-150; BUY-110 does not reintroduce local fake success for those CTAs.
