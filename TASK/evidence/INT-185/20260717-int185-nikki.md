## Multi-replica safety

1. **Job cadence exclusivity:** `job_leases` row per job name; `INSERT … ON CONFLICT DO UPDATE WHERE lease_until <= now OR owner = self`. Only one replica runs each job while lease is live.
2. **Row batch exclusivity:** lifecycle SQL uses `FOR UPDATE SKIP LOCKED` (CTE updates/deletes) or service-level idempotent transitions.
3. **Outbox workers:** existing `SKIP LOCKED` / lease columns on `outbox_events` retained.

## Graceful drain

On `ctx` cancel: `drainMode` stops new leases; runner waits up to 2m for in-flight batches; success/failure releases lease (`lease_until = now`).

## Clock

`ports.Clock` injected into `LeaseStore` and `Runner` (composition uses `SystemClock`; tests use fixed clocks).

## Ownership boundary (API vs worker)

| Transition | Owner |
| --- | --- |
| User/API expire, cancel, paid callback | API / callback path (sync) |
| TTL elapsed intents, unknown lookup, reservation expiry, quote expiry, retention | Worker jobs only |

Duplicate worker runs must not double-release stock/coupon, double-post ledger, or double-send mail (domain services already idempotent; lease prevents concurrent job entry).
