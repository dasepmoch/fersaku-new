## 3. Endpoints

| Method | Path | Permission | Notes |
| --- | --- | --- | --- |
| GET | `/v1/admin/reviews` | `reviews.read` | Bounded first result (limit ≤100 default 50); no paging UI |
| GET | `/v1/admin/reviews/{reviewId}` | `reviews.read` | Detail mapper |
| POST | `/v1/admin/reviews/{reviewId}/transition` | `reviews.moderate` | Body: `status` + `reason` (≥12); status `PUBLISHED\|NEEDS_EDIT\|REMOVED\|PENDING` |

BE does not require `expectedVersion` on this transition; noop same-status is accepted with audit. ControlDialog keeps reason on failure (no success path / dialog input preserved until close).
