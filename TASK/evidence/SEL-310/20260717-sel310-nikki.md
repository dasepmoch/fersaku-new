## 4. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Domain list (api) | GET domains | Map status/TLS → chip + detail line |
| Create | POST | Token component-local only; stripped from list model |
| Verify | POST verify | Idempotent path when BE returns ACTIVE |
| Delete | DELETE | Tombstone/cooldown on BE |
| Card chrome | LinksPanel | Same geometry; Connected only when ACTIVE+TLS ACTIVE |
| Empty API | — | `No custom domain · platform host only` / `Not connected` |
| Mock | fixtures | `shop.asep.ai • DNS verified` / Connected |
| SEO/name/links | SEL-300 | storefront draft `seoTitle`/`seoDescription`/name/bio/customLinks |
| Support presentation | PATCH store | Impersonation allowlist only (name/description) |
| Live custom domain | **not rolled out** | BE `dnsadapter.NewFake` + `edgeadapter.NewFake` |

### Security

- `verificationToken` never in query keys or list cache.
- Mapper rejects `verificationTokenHash` leakage.
- Domain create hostname validation remains BE-authoritative (IDNA/public suffix).
