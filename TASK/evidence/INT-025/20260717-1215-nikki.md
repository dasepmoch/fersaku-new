## 1. Objective delivered

Replace global `isLiveApi()` as the rollout decision path with a typed per-domain registry (`mock | api | disabled`), request-stable SSR→client snapshot, fail-closed live semantics, and architecture guards.

| Concern | Module |
| --- | --- |
| Types + pure evaluation + adapters helpers | `shared/data/domain-source.ts` |
| SSR/client snapshot install (non-sensitive) | `shared/data/domain-source-provider.tsx` |
| Root request-stable evaluation + provider | `app/layout.tsx` |
| Deprecated global switch | `shared/data/mode.ts` (`isLiveApi` kept only as legacy export; no feature imports) |
| Unit tests | `tests/unit/domain-source.test.ts` |
| Architecture forbid `isLiveApi` / env reads in presentation | `tests/unit/architecture-boundaries.test.ts` |

## 2. Domain model

```ts
type DomainSource = "mock" | "api" | "disabled";
type DataDomain =
  | "publicCatalog"
  | "auth"
  | "checkout"
  | "buyer"
  | "sellerCatalog"
  | "sellerOperations"
  | "sellerFinance"
  | "adminRead"
  | "adminWrite";
```

Money/privileged surfaces (`sellerFinance`, `adminWrite`, `checkout`) are independent keys so public reads can stay on `api` while those domains are `disabled`.

## 3. Evaluation rules

| Stage / input | Behavior |
| --- | --- |
| prototype + bootstrap `mock` | All domains `mock` unless overridden |
| prototype + bootstrap `api` | All domains `api` unless overridden |
| Per-domain override | `DOMAIN_SOURCE_OVERRIDES` JSON (server-owned, not `NEXT_PUBLIC_*`) |
| live + residual `mock` | Rewritten to `disabled` (never fixtures) |
| live + bootstrap `mock` without explicit safe map | `DomainSourceConfigError` fail closed |
| `assertProductionDomainSources` | Throws if any domain still `mock` |
| `shouldUseMockFixtures(domain)` | `true` only for `mock`; throws `DomainDisabledError` for `disabled`; `false` for `api` |
| `mockPlaceholderData` | Returns fixtures only when source is `mock`; never for `disabled`/`api` |

Actor/tenant permissions remain backend session/bootstrap (INT-120/150); this registry is source selection only.

## 4. Adapter injection pattern

Feature `api.ts` / data modules:

```ts
import { shouldUseMockFixtures } from "@/shared/data/domain-source";

if (shouldUseMockFixtures("publicCatalog")) return demo…;
// else api transport
```

Feature hooks:

```ts
import { mockPlaceholderData } from "@/shared/data/domain-source";

placeholderData: mockPlaceholderData("buyer", demo…),
```

Migrated call sites (no remaining `isLiveApi` outside `shared/data/mode.ts`):

- `features/catalog/{api,hooks}.ts` — `publicCatalog` + `sellerCatalog`
- `features/commerce/checkout/api.ts` — `checkout`
- `features/buyer/data/{api,hooks}.ts` — `buyer`
- `features/finance/{api,hooks}.ts` — `sellerFinance`
- `features/orders/{api,hooks}.ts` — `sellerOperations`
- `features/seller/{customers,inventory,reviews,storefront}/*` — catalog/operations/public as appropriate
- `features/admin/data/*` — `adminRead` / `adminWrite`

## 5. SSR / hydration snapshot

1. Server: `createBootstrapDomainSourceSnapshot()` once in root layout.
2. `toPublicDomainSourceSnapshot()` strips nothing secret (map is non-sensitive by design).
3. `DomainSourceProvider` installs the same map on the client before children read `getDomainSource`.
4. Snapshot includes `version`, `releaseId`, `stage`, `domains` for cache/telemetry keys (QLT-400 can extend).

Emergency control: `DOMAIN_SOURCE_OVERRIDES` + optional `DOMAIN_SOURCE_VERSION` / `DOMAIN_SOURCE_RELEASE_ID` (server env). Build-time `NEXT_PUBLIC_DATA_SOURCE` remains bootstrap default only until QLT-420 deprecates it.

## 6. Disabled UX disposition

- Adapters throw `DomainDisabledError` (`code: DOMAIN_DISABLED`) — **no mock row/success/secret**.
- User-visible surface reuses existing route error/permission/disabled controls (UXE-004 / route-state matrix); no new maintenance banner invented in this task.

## 7. Architecture rules

- New code must not call `isLiveApi()` or import `@/shared/data/mode` outside the registry module.
- Screens/components/hooks must not read `NEXT_PUBLIC_DATA_SOURCE` / `publicEnv.dataSource` for source selection.
