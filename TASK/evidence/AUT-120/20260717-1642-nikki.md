## 3. Behavior

| Ceremony | Transport | Notes |
| --- | --- | --- |
| Password forgot | `POST /v1/auth/password/forgot` | Already AUT-100; generic anti-enumeration |
| Password reset | Fragment `#token=` → scrub → `POST /v1/auth/password/reset` | Compose on `/login` only |
| Password change | `POST /v1/auth/password/change` | Adapter ready; rotates CSRF when returned |
| Email change request | `POST /v1/auth/email-change/request` | Adapter only |
| Email confirm current/new | Fragment → scrub → confirm-* POST | On complete: `logoutSession({ redirect: false })` |
| MFA verify (login) | `POST /v1/auth/mfa/verify` | Seller + admin login shells; optional recent proof mint |
| MFA step-up | `POST /v1/auth/mfa/step-up` | Stores proof via INT-140 `setRecentMfaProof` |
| MFA enroll | `POST /v1/auth/mfa/enroll` | Secret + otpauth once, component memory |
| MFA confirm | `POST /v1/auth/mfa/confirm` | Recovery codes once |
| MFA disable / regen | `POST .../disable`, `.../recovery-codes/regenerate` | Require fresh code |
| Verify email | `POST /v1/auth/verify-email` | Shared fragment ceremony helper |

### Error disposition

| Case | Mapping |
| --- | --- |
| Invalid reset/email/verify token | `invalid_token` (no account existence leak) |
| Invalid MFA code | `invalid_code` on existing field region |
| Rate-limit / unavailable | `blocked` — no fake success (UXE-011) |
| Email change 409 | `conflict` |

### Security

- Bootstrap tokens only in POST body after fragment scrub
- Mutation keys secret-free; `gcTime: 0`
- Recovery codes / TOTP secret never in RQ cache / web storage
- Recent proof: INT-140 memory only + `X-Recent-MFA-Proof` header path
