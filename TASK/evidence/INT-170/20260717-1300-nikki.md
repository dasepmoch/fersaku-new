## 3. Policies (summary)

### Mock interaction

- Feedback enabled only when **all** domains are `mock` and stage â‰  `live`
- Any domain `api` â†’ no mock toast (shell markup unchanged)

### No silent mock fallback

- Adapters use `shouldUseMockFixtures` **before** transport only
- `rejectAsApiFailure` / `mustNotTreatAsEmpty` / `isEmptyOrMock: false` prevent empty/mock success
- Unit test scans representative adapters for `catch â†’ demo/mock` patterns

### Architecture reachability

- Presentation (app/components/feature screens/ui/domains) must not resolve-import `features/**/mock.ts` or `mock-audit.ts`
- App pages using `DEMO_STORE_ID` must also gate with domain-source
- Exemptions: adapters (`/api.ts`, `/data/`, hooks), tests, shared mock runtime, api playground, shells still hosting prototype impersonation chrome (audit now via `client-audit` no-op in API mode)

### Redaction + reporter

- Keys: cookie, token, CSRF, MFA, Authorization, API keys, QR/qris, email/phone, bank, signed URL, inventory/delivery secret, response/request body
- Nested arrays/objects + Error `cause` chains
- Schema errors: issues only â€” no raw body in telemetry
- Metrics: `http.client.error|contract_invalid|session_expired|latency_ms|retry` (bounded names)

### Error presentation

- `presentClassifiedError` / `presentThrownError` â†’ existing UI-050 surfaces
