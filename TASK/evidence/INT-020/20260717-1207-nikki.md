## 2. Error mapping policy (code-enforced)

| HTTP/problem | `classifyApiError` kind | Guarantees |
| --- | --- | --- |
| 400 malformed | `form_generic` | `isEmptyOrMock: false` |
| 400 `VALIDATION_FAILED` | `form_field_violations` | typed `fieldViolations` |
| 401 | `session_expired` | no safe-get auto retry as success |
| 403 CSRF (`AUTH_CSRF_INVALID`) | `csrf_recovery` | mutation must reuse same idempotency key (policy) |
| 403 permission | `permission_denied` | no retry |
| 404 `RESOURCE_NOT_FOUND` | `resource_not_found` | `mayMapToNull` only for that code |
| 409 conflict | `conflict_preserve_draft` | no mutation auto-retry |
| 409 `IDEMPOTENCY_CONFLICT` | `idempotency_conflict` | `autoRotateKey: false` |
| 429 | `rate_limited` | `retryAfterSeconds`; safe GET only |
| 5xx / network / timeout | `retry_safe_get` / `transport_failure` | mutation `mutationAutoRetry: false` |

Helpers that prevent empty/mock masking:

- `mustNotTreatAsEmpty` — always true for classified failures
- `rejectAsApiFailure` — never returns `[]`
- `mapDetailOrThrow` — null only for expected resource-not-found

## 3. Pagination

- `numberedMetaToTablePagination` → `{ page, pageSize, total, pageCount, start, end }` matching existing `TablePagination` / `ClientPagination` fields without visual change.
- Cursor helpers expose `hasMore` / opaque cursors only; no total/page jump invented from history.
- Authoritative `totalCount`/`pageCount` from NumberedPageListMeta (not `data.length` alone).

## 4. Idempotency

- `createIdempotencyKey()` / `beginIdempotencyIntent()` — opaque UUID per logical intent
- `fingerprintBody` order-stable; same key + different body → local `conflict_local` (mirrors 409)
- `requireNewIntentAfterIdempotencyConflict()` — no automatic key rotation

- No BE code change required (codes already stable); optional domain codes remain BE-owned.
