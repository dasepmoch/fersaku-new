## 3. Behavior

| Surface | Transport | Mapping |
| --- | --- | --- |
| Featured | `GET /v1/public/products/featured` | DTO requires `storeSlug` → `FeaturedCatalogProduct[]` |
| Storefront | `GET /v1/public/stores/{slug}` | Schema + map; products get storeSlug; 404 → null |
| Product | `GET /v1/public/products/{idOrSlug}?store=` | Store-bound when storeSlug known; 404 → null |
| Reviews | `GET /v1/public/products/{id}/reviews` | Cursor list → card model; 404 → [] |
| Summary | `GET /v1/public/products/{id}/reviews/summary` | `count/averageRating/rating1..5` → `total/average/distribution` |

### 404 vs network

- `RESOURCE_NOT_FOUND` / classified `resource_not_found` → `null` (store/product) or empty reviews/summary 
- Network / 5xx / other → rethrow (pages surface via error boundary, not fake not-found)

### Featured storeSlug

- Homepage: `href={publicProductHref(p.storeSlug, p.slug)}` — never hardcode `@asep-ai-tools` for all merchants 
- Static demo CTA “Lihat toko demo” remains demo-only (not featured product loop)

### Zero-review

- `mapPublicReviewSummaryDto({ count: 0, ... })` → `total=0`, `average=0`, distribution zeros 
- `reviewDistributionWidthPercent` returns `0` when total=0 (no `NaN%`)

### BE

- Featured SQL: `products` ⨝ `stores` where product `published` and store `ACTIVE` 
- Public product by id/slug also requires ACTIVE store 
- `GetPublishedProductByStoreAndSlug` for dual-store same product slug 
- Response includes `storeSlug` when known
