## Scope delivered

### Backend
- Customer = store-scoped aggregate derived from `orders` (not global buyer profile)
- Stable `customerId` = `sha256(store_id || unit-separator || lower(trim(buyer_email)))` hex
- Guest/empty-email rows excluded; merge key is normalized email within store
- `GET /v1/stores/{storeId}/customers` — NumberedPageList (`page`/`pageSize`/`totalCount`/`pageCount`), search `q`, stable sort `last_purchase DESC, customer_id DESC`, max pageSize 50
- `GET /v1/stores/{storeId}/customers/{customerId}` — metrics, bounded order history (20), optional note, consent placeholder (`UNKNOWN` — no invented channel)
- `PUT /v1/stores/{storeId}/customers/{customerId}/notes` — versioned upsert; conflict on version mismatch
- Migration `000032_seller_customers` → `store_customer_notes`
- Store membership + platform admin; foreign store/customer → safe `RESOURCE_NOT_FOUND`
- OpenAPI paths + schemas; router inventory updated
- Export CSV / Kirim email remain unwired (no characterized action beyond existing inert buttons)

### Frontend
- Schemas: `sellerCustomer*DtoSchema` / list+detail+note envelopes in `shared/api/schemas.ts`
- Mappers: `features/seller/customers/mappers.ts` → existing `SellerCustomer` view (server customer id, not order id)
- API: `listSellerCustomers` / `getSellerCustomer` / `upsertSellerCustomerNote`
- Hooks: store-scoped query keys with filters; note mutation invalidates list/detail (no optimistic success)
- SSR: `features/seller/customers/server.ts` via `serverApiRequest` + cookie forward + no-store; foreign → `notFound()`
- Screen: keep list/detail geometry; server metrics/history/notes; search + TablePagination server page
- Mock path preserved for prototype
