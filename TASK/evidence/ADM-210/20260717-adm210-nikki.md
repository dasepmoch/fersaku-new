## 1. Objective delivered

- Wire admin buyer list/detail when `adminRead=api` (ADM-120 foundation)
- Purchases/sessions composition from server; no delivery secrets
- Fix session async local-state copy bug → authoritative query only
- Support actions (magic-link / email-change / session revoke) via existing ControlDialog + reason + auth gate

- Unit tests: list, detail, permission deny, mock path, no secrets

## 3. Backend contract (consumed)

| Endpoint | Permission (BE) | Use |
| --- | --- | --- |
| `GET /v1/admin/buyers` | `buyers.read` | Bounded list |
| `GET /v1/admin/buyers/{buyerId}` | `buyers.read` | Detail |
| `GET /v1/admin/buyers/{buyerId}/purchases` | `buyers.read` | Purchase shells (no delivery secret) |
| `GET /v1/admin/buyers/{buyerId}/sessions` | `buyers.read` | Session metadata (hashed IP) |
| `POST /v1/admin/actions` | `merchants.write` (BE today) | `buyer.magic_link.send`, `buyer.email_change.start`, `buyer.sessions.revoke` |

Domain sources: `adminRead` (reads), `adminWrite` (support commands).
