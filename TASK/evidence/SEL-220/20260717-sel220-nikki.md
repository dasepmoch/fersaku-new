## 3. Backend contract (consumed)

| Endpoint | Use |
| --- | --- |
| `POST /v1/stores/{storeId}/products` | Create draft; `CreateProductRequest`; 201 envelope |
| `PATCH /v1/stores/{storeId}/products/{productId}` | Patch fields (not status) |
| `POST /v1/stores/{storeId}/products/{productId}/publish` | Catalog publish (empty body) |
| `POST /v1/stores/{storeId}/products/{productId}/archive` | Archive; product envelope |
| `GET /v1/stores/{storeId}/products/{productId}` | Detail (existing; 404 → null) |

Domain source: `sellerCatalog`.

**Delivery freeze:** wire `type` enum remains `download | link | code` only. 
`credentials` is a UI delivery option mapped client-side; inventory schema (SEL-240) owns CREDENTIAL fields.

**Release vs publish:** Delivery-tab “Publish update to existing buyers” does **not** call catalog publish in API mode (no release endpoint; SEL-230). Catalog publish is draft→published via create+publish or save on draft detail.
