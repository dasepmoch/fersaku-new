## 3. Backend contract (consumed)

| Endpoint | Permission (BE) | Use |
| --- | --- | --- |
| `GET /v1/admin/orders` | `orders.read` | Bounded list |
| `GET /v1/admin/orders/{orderId}` | `orders.read` | Detail projection |
| `GET /v1/admin/payments` | `payments.read` | List; source STOREFRONT\|QRIS_API only |
| `GET /v1/admin/payments/{paymentIntentId}` | `payments.read` | Detail |
| `GET /v1/admin/payment-mismatches` | `payments.read` | Read-only mismatch evidence |
| `POST /v1/admin/orders/{orderId}/delivery/resend` | `fulfillment.force` | Resend only (reason + idempotency) |
| `POST /v1/admin/payments/{paymentIntentId}/provider-lookup` | `payments.read` | Lookup accepted; no client status |

Domain sources: `adminRead` (reads), `adminWrite` (commands).

Force-fulfill/revoke: **not** wired on order detail (ADM-320 fulfillment composition only).
