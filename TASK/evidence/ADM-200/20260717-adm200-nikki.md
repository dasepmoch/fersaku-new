# ADM-200 — Merchant list/detail, status, API capability, credential support

**Status:** `[x]` completed  
**Actor:** @nikki/opencode  
**When:** 2026-07-17  
**Depends on:** ADM-110, ADM-120, SEL-100 (all `[x]`)  
**Branch:** main

## 1. Objective delivered

- Wire admin merchant list/detail when `adminRead=api` (ADM-120 foundation)
- Typed merchant status + QRIS API access mutations (independent axes)
- Credential authorize/rotate support without raw key to admin
- Finance projection for detail metrics (server money authority)
- Permission gates: `merchants.read` / `merchants.write` (ADM-110)
- Unit tests: list, detail, permission deny, mock path, typed commands

## 3. Backend contract (consumed)

| Endpoint | Permission (BE) | Use |
| --- | --- | --- |
| `GET /v1/admin/merchants` | `merchants.read` | Bounded list |
| `GET /v1/admin/merchants/{id}` | `merchants.read` | Detail |
| `GET /v1/admin/merchants/{id}/finance/summary` | `merchants.read` | Detail money metrics |
| `POST /v1/admin/merchants/{id}/status` | `merchants.write` | Lifecycle ACTIVE\|SUSPENDED\|CLOSED |
| `POST /v1/admin/merchants/{id}/api-access/status` | `merchants.write` | QRIS API ACTIVE\|SUSPENDED |
| `GET /v1/admin/merchants/{id}/api-credentials` | `kyc.review` (BE) | Masked list only |
| `POST .../api-credentials/rotate` | `kyc.review` (BE) | Authorize rotation metadata (no raw key) |

Domain sources: `adminRead` (reads), `adminWrite` (commands).

## 4. Behavior

| Path | Behavior |
| --- | --- |
| Mock | Fixtures only; no transport; prototype labels preserved where needed |
| API + `merchants.read` | List/detail/finance schema-mapped |
| API without read claim | Query disabled (fail-closed) |
| Status vs API access | Independent; typed POSTs with reason + MFA + idempotency |
| Credential rotate | No optimistic raw key; schema rejects `fsk_live_`/`fsk_test_` |
| After mutation | Invalidate merchant/list/finance/credentials/audit keys |
| Money | Finance amounts server-only; no client credit/debit |
