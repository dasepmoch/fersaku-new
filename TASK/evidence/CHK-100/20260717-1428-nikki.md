## 1. Objective delivered

Checkout resolves product/store via public catalog API (no hardcoded product/order). Server `POST /v1/checkout/quote` is the money authority: request carries identifiers/selections only; UI displays mapped `merchandise` / tip / upsell / `gross`. Debounced re-quote with abort + sequence guard so stale responses never overwrite. Coupon UI remains DISABLED/OUT-OF-SCOPE. QR/wallet/pay semantics frozen for CHK-110. Browser never calls stock-reservations.

## 3. Behavior

| Surface | Transport | Mapping |
| --- | --- | --- |
| Bootstrap | `getPublicProduct` / `findPublicProduct` + `getPublicStorefront` | product + store; `storeId` for quote |
| Quote | `POST /v1/checkout/quote` | identifiers + tip/upsell/PWYW merchandise only → `CheckoutQuote.gross` |
| Display | existing summary/details/qris props | `base`/`tip`/`upsell`/`total` from quote when present |
| Mock | `shouldUseMockFixtures("checkout")` | local integer math; `clientDiscountIgnored` |

### Authority rules

- Browser total is **never** authority; only `quote.gross` (and mapped lines) after server response.
- Optional `clientDiscount` may be sent only to prove ignore; BE sets `clientDiscountIgnored: true`.
- Coupon code omitted (no UI control). Disposition: **DISABLED/OUT-OF-SCOPE** until UI-080.

### Debounce / stale

- `useCheckoutQuote`: 280ms debounce, `AbortController` cancel, monotonic sequence — older responses discarded.

### QR / wallet freeze (`CHECKOUT_QR_WALLET_SEMANTICS`)

| Control | Disposition |
| --- | --- |
| Continue (details) | PRESENTATIONAL step only — does **not** create intent |
| Pay | Mock/local simulate; live → CHK-110 `createCheckoutIntent` |
| Wallet chips | PRESENTATIONAL until provider deep-link contract |
| QR display | PRESENTATIONAL until intent payload; no copy control → no new button |
| Coupon | DISABLED/OUT-OF-SCOPE |
| Stock reservations | INTERNAL_ONLY — FE never calls `POST /v1/checkout/stock-reservations` |

### Stock-reservations disposition

- Endpoint remains on BE router as foundation for create-intent (CHK-110).
- **Browser adapters do not expose or call it.**
- Documented in contracts + evidence; create-intent must own reservation internally.

### BE public `storeId`

- `productDTO` always includes `storeId` when known (needed for quote).
- `publicStorefrontDTO` surfaces `storeId` from first product with StoreID.
- Seller-only fields (`merchantId`, `status`) remain seller-gated.
