## 1. Objective delivered

Single browser transport with nested ProblemEnvelope parse, OpenAPI header names, abort/timeout combine, schema gate, CSRF session hooks, 401 dedupe, and redacted diagnostics.

| Concern | Module |
| --- | --- |
| Browser `apiRequest` + URL builder + session hooks | `shared/api/http-client.ts` |

| Unit + architecture gate | `tests/unit/http-client.test.ts`, `architecture-boundaries.test.ts` |

## 2. Checklist FE

- [x] Parse `{problem:{...}}`, not top-level problem (legacy top-level still accepted as fallback)
- [x] Preserve status, stable code, details/field violations, request ID, `Retry-After`
- [x] Header standardized to `session authentication` (was `X-Recent-auth`)
- [x] Relative `/v1` same-origin (INT-030 preserved)
- [x] Content-Type + JSON validation; empty `204`, invalid JSON, invalid schema distinguished
- [x] Caller abort + timeout combined without listener leak
- [x] Request ID cryptographically random when available; redacted reporter context only
- [x] CSRF auto via `setHttpClientSessionHooks({ getCsrfToken })` for unsafe methods; explicit `csrfToken` still works (INT-120/130 extension points)
- [x] Sensitive context (audit/idempotency/If-Match) only when caller opts in
- [x] No body/header secret/full response in reporter
- [x] No auto-retry mutations in client (`apiRequest` is single-shot; query mutation retry remains false)
- [x] Safe GET retry remains query-policy concern (documented; client does not auto-retry)
- [x] 401 session-expired handling deduped via hooks (`onSessionExpired`)

## 3. Schema enforcement (phased)

- Runtime: `requireSchema` defaults **true** — missing schema on non-204 success → `INVALID_API_CONTRACT`
- Architecture test: every `features/**` `apiRequest` call must include `schema:`
- Catalog public reads use domain DTO envelopes; other live adapters use `structuralEnvelopeSchema` / `structuralCursorPageEnvelopeSchema` until domain DTO mappers land
- Tests may use `requireSchema: false` for transport-only cases
