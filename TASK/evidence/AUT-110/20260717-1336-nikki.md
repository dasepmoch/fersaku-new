## 1. Objective delivered

Wire buyer `/account/login` + `/account/verify` to real magic-link transport when domain `auth` is `api`; keep mock fixtures when `auth` is `mock`. Token delivery is URL **fragment only** (`#token=`), scrubbed before POST; consume body is one-time; success bootstraps session + CSRF + safe `returnTo`.

## 3. Behavior

| Action | Transport | Post-success / failure |
| --- | --- | --- |
| Request | `POST /v1/auth/magic-link/request` `{ email }` | Always generic success when 200 (anti-enumeration). Rate-limit/unavailable → `blocked` (stay form, no fake sent; UXE-011) |
| Consume | `POST /v1/auth/magic-link/consume` `{ token }` | `applyLoginCsrf` + `refreshSessionAfterLogin("buyer")`; redirect safe `returnTo` or `/account/purchases` |
| Invalid/expired/replay | 401 problem | `invalid_token` → existing `notFound()` — no success, no ad-hoc error card |
| Query/path token | Forbidden | Detected; never consumed; `notFound()` |
| Mock request | no network | generic_sent; prototype mock link `#token=mock_buyer_token` only when `auth=mock` |
| Mock consume | no network | buyer session bootstrap + redirect |

### Security

- Token only from `location.hash` (`parseMagicLinkFragmentToken`); immediate `history.replaceState` scrub (`scrubUrlFragment`) before network
- Token only in POST body; mutation keys `["auth","buyer","magic-link",…]` never include token values; `gcTime: 0`
- No password on buyer surface; `objectContainsMagicTokenLeak` unit coverage
- `returnTo` via `sanitizeReturnToForSurface("buyer")` only
- Anonymous POSTs use `skipCsrf: true` (BE allows unresolved cookie on magic-link)
