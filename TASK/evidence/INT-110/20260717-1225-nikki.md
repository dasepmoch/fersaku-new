## 1. Objective delivered

Server-only transport that calls `API_INTERNAL_URL` with an explicit allowlisted `Cookie` + `X-Request-ID` from the incoming Next request. Browser `apiRequest` (`credentials: include`) remains the client path and is **not** the sole SSR auth mechanism.

| Concern | Module |
| --- | --- |
| Pure cookie/header allowlist helpers | `shared/api/server-cookie-forward.ts` |
| Server `serverApiRequest` + Next cookies/headers | `shared/api/server-http-client.ts` (`import "server-only"`) |
| Private SSR architecture policy | `shared/api/SERVER-SSR-POLICY.md` |
| Unit tests | `tests/unit/server-cookie-forward.test.ts`, `server-http-client.test.ts` |
| Architecture gate | `tests/unit/architecture-boundaries.test.ts` (INT-110) |

## 2. Topology (SSR path)

```text
Browser request (Cookie: fersaku_session=…)
  → Next Server Component
      serverApiRequest("/v1/…")
        → requireApiInternalUrl()  // server-only, not NEXT_PUBLIC
        → Cookie: fersaku_session only (allowlist)
        → X-Request-ID forwarded when present
        → cache: "no-store" for privacy: "private"
        → fetch → http://api:8080 (or configured internal)
```

Browser path unchanged (INT-100): relative `/v1` + `credentials: "include"`.
