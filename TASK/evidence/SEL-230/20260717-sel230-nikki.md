## 3. Backend contract (consumed)

| Endpoint | Use |
| --- | --- |
| `POST /v1/stores/{storeId}/objects/uploads` | Upload intent + short-lived `uploadUrl` |
| `PUT` (presigned) | Direct bytes to object storage (not via Next) |
| `POST /v1/stores/{storeId}/objects/{objectId}/complete` | Checksum verify → READY/REJECTED |
| `GET /v1/stores/{storeId}/objects/{objectId}` | Metadata poll (no raw storage key) |

Domain source: `sellerCatalog`.

**Purposes used:** `PRODUCT_FILE` on product delivery controls. `PUBLIC_ASSET` available on adapters for store branding (SEL-310). Never `PROFILE_ASSET` for personal avatar.
