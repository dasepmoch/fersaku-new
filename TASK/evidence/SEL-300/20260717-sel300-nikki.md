## 4. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Load (api) | GET studio | Map `draftConfig` → `BuilderConfig`; revision/etag authority |
| Autosave | PUT draft (800ms debounce) | Coalesced; abort prior; no keystroke spam |
| Undo/redo | client-only | Presentation history; not server authority |
| 409 | throw + status line | Local config kept; server revision/etag updated from details |
| Publish | flush dirty draft then POST | Opaque idempotency UUID; no optimistic “Published” |
| Preview products | `useSellerProducts` | API path; mock uses demo fixtures |
| Mock | localStorage draft | revision 14; no network |
| Invalidation | storefront (+ products list) | After draft/publish success |

### Wire body

- Root: only `config`, `expectedRevision`, `expectedETag` (publish may omit config after flush).
- No FE view fields (`storeId`, `reason`, `idempotencyKey`, root `logoStyle`) on body.
- `logoStyle` nested under `config` for round-trip; public merge ignores unknown keys.

### BE

- Publish handler: `ContentLength != 0` → strict `DecodeJSON` (unknown fields fail).
- Existing draft overwrite + publish revision/ETag conflict + next draft shell unchanged.
