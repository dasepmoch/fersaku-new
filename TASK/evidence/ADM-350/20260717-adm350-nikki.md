## 1. Objective delivered

Wire admin dual-resource webhook console on `/admin/webhooks` when admin domain is `api`:

- Inbound **provider callbacks** (`GET/POST …/provider-callbacks`)
- Outbound **seller webhook deliveries** (`GET/POST …/seller-webhook-deliveries`)

Discriminated union rows, stable `kind:id` keys, separate replay/retry permissions, redacted payload evidence only, partial-source failure surface, force-fulfill via order delivery command (ADM-320). No UI redesign.

## 3. Endpoints

| Method | Path | Permission | Notes |
| --- | --- | --- | --- |
| GET | `/v1/admin/provider-callbacks` | `webhooks.read` | Bounded list (BE limit 50); no encrypted payload |
| GET | `/v1/admin/provider-callbacks/{callbackId}` | `webhooks.read` | Inbound only; rejects seller IDs |
| POST | `/v1/admin/provider-callbacks/{callbackId}/replay` | `provider_callbacks.replay` | Body `{ reason }`; Idempotency-Key + audit reason + recent MFA |
| GET | `/v1/admin/seller-webhook-deliveries` | `webhooks.read` | Optional `status`, `merchantId`; outbound only |
| GET | `/v1/admin/seller-webhook-deliveries/{deliveryId}` | `webhooks.read` | Rejects provider callback IDs |
| POST | `/v1/admin/seller-webhook-deliveries/{deliveryId}/retry` | `seller_webhook_deliveries.retry` | Body `{ reason }`; MFA + idempotency |
| POST | `/v1/admin/orders/{orderId}/delivery/force-fulfill` | `fulfillment.force` | Used by Force-Fulfill dialog when verified evidence bound |

## 4. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Table compose (api) | Parallel GET both lists | `Promise.allSettled`; partial error banner |
| Row key | `PROVIDER_CALLBACK:{id}` / `SELLER_DELIVERY:{id}` | Prevents cross-namespace collisions |
| Replay (api) | POST provider-callbacks replay | Only failed callback rows; permission-gated |
| Retry (api) | POST seller-webhook-deliveries retry | Only failed seller rows; never Xendit path |
| Force-fulfill (api) | POST order delivery force-fulfill | Evidence gate unchanged; no local fake update |
| Mock | `initialWebhooks` seed | Local optimistic retry + client audit |
| Secrets | Schema superRefine | Reject `encryptedPayload`, `signingSecret`, `signature`, etc. |

### Redaction policy

- Callback list/detail: metadata + `payloadDigest` ref only (`digest:…` / `redacted`).
- Seller detail: endpoint host + payload hash prefix; response body label remains redacted.
- Payload preview stays structural placeholders (order/status/reference) — no raw body.
