## 1. Objective delivered

Wire storefront product search on the existing search chrome and sanitize Instagram/YouTube/website socials so live storefront never keeps no-op `href="#"` or unsafe schemes. Filter already-bounded published products client-side (no public search endpoint; tenant fixed by store DTO). No UI redesign.

## 3. Behavior

| Surface | Behavior |
| --- | --- |
| Search button | Existing header control toggles inline search input; filters `store.products` via `filterStorefrontProducts` (title/slug/short); empty query shows full bounded list |
| Product links | `publicProductHref(store.slug, p.slug)` — canonical store slug from same DTO |
| Instagram | Bare handle → `https://www.instagram.com/{handle}/`; full URL only on instagram.com hosts |
| YouTube | Bare handle → `https://www.youtube.com/@{handle}`; allowlisted youtube.com / youtu.be |
| Website | `https` only after normalize; bare host gets `https://`; missing/unsafe → omit icon |
| External attrs | `rel="noopener noreferrer"` + `target="_blank"` |
| Rejected | `javascript:`, `data:`, `http:`, credential URLs, wrong hosts, `#` |

### Public catalog mode

- Mock and API both go through sanitized social mapping (DTO mapper / demo mapper).
- Search filters the products already returned by `getPublicStorefront` (no inventing a public search API).
