## 3. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Buyer access (api) | `POST /v1/buyer/purchases/{orderId}/delivery/access` | Session cookie; secrets only in response; 404 → null |
| Order access (api) | `POST /v1/orders/{orderId}/delivery/access` | Session and/or body `token`; never query |
| Buyer resend (api) | `POST /v1/buyer/purchases/{orderId}/delivery/resend` | Idempotency-Key; queued; no secrets |
| Mock buyer/checkout | adapter fixtures | Deterministic claim; no network |
| Base purchase/order | GET (CHK-130 / BUY-100) | Redacted shells only — unchanged |

### Secret lifecycle

- Claim only after explicit existing CTA.
- Secrets in component state/ref only; cleared on unmount, order remount key, visibility+TTL.
- Signed URL (if present in secrets) opened with `window.open`; not logged/persisted.
- `downloadObjectId` opaque — does not invent seller-scoped download-url.

### Fail closed

- Foreign buyer → safe 404 null.
- Order access denied/revoked/expired → null (no fake success).
- 401 rethrows for auth flow.
