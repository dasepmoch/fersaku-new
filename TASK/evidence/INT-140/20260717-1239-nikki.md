## 1. Design (fail-closed)

### authenticated session session gate

- Login with logined still issues a session cookie (session verification timestamp null).
- Global middleware session gate (legacy name) (after session load, before CSRF/business routes):
 - Condition: `session authenticated (auth unused)`
 - **Allowlist only:**
 - `GET /v1/auth/session`
 - `POST /v1/auth/auth/verify`
 - `POST /v1/auth/logout`
 - `POST /v1/auth/sessions/revoke-all`
 - `POST /v1/auth/password/forgot`
 - `POST /v1/auth/password/reset`
 - All other routes → `401 AUTH_AUTH_REQUIRED` (roles/permissions ignored).
- `GET /session` while pending returns `sessionStatus: "authenticated session"` and **empty** `permissions`/`roles`.

### Recent authentication proof (`session authentication`)

| Property | Value |
| --- | --- |
| Header | `session authentication` (opaque; never password/seed) |
| Mint | `POST /v1/auth/auth/verify` with optional `purpose`, or `POST /v1/auth/auth/step-up` (purpose required) |
| Factors | `password`, `recovery`, `password` (non-auth accounts for step-up only) |
| Purposes | `inventory.reveal`, `credentials.rotate`, `bank.change`, `withdrawal.create`, `admin.command` |
| TTL | 5 minutes (`auth.RecentProofTTL`) |
| Use | single-use consume; hash at rest in `mfa_recent_proofs` |
| Bind | user_id + session_id + purpose |
| Revoke | logout revokes open proofs for session |

Response mint metadata (once):

```json
{
 "mfaVerified": true,
 "recentMfaProof": "<opaque>",
 "purpose": "inventory.reveal",
 "expiresAt": "2026-07-17T12:44:00Z",
 "factor": "password"
}
```

Problem codes:

- `AUTH_AUTH_REQUIRED` — pending session or missing proof
- `AUTH_MFA_PROOF_INVALID` — wrong session/purpose/replay/unknown
- `AUTH_MFA_PROOF_EXPIRED` — TTL elapsed

### Consumer wiring

- Inventory reveal: `RequireRecentMFAProof(inventory.reveal)` + body `mfaVerified` **ignored** (no authority).
- Helper `middleware.RequireRecentMFAProof(purpose, validator)` ready for credentials/bank/withdrawal/admin.

### Deferred (explicit)

- Login for invited admin without factor → **ADM-100** (login still blocks admin without logined).
