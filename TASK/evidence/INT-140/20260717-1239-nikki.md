## 1. Design (fail-closed)

### MFA_PENDING session gate

- Login with MFA enrolled still issues a session cookie (`mfa_verified_at` null).
- Global middleware `MFAPendingGate` (after session load, before CSRF/business routes):
  - Condition: `MFAEnabled && !MFAVerified`
  - **Allowlist only:**
    - `GET /v1/auth/session`
    - `POST /v1/auth/mfa/verify`
    - `POST /v1/auth/logout`
    - `POST /v1/auth/sessions/revoke-all`
    - `POST /v1/auth/password/forgot`
    - `POST /v1/auth/password/reset`
  - All other routes → `401 AUTH_MFA_REQUIRED` (roles/permissions ignored).
- `GET /session` while pending returns `sessionStatus: "MFA_PENDING"` and **empty** `permissions`/`roles`.

### Recent MFA proof (`X-Recent-MFA-Proof`)

| Property | Value |
| --- | --- |
| Header | `X-Recent-MFA-Proof` (opaque; never TOTP/seed) |
| Mint | `POST /v1/auth/mfa/verify` with optional `purpose`, or `POST /v1/auth/mfa/step-up` (purpose required) |
| Factors | `totp`, `recovery`, `password` (non-MFA accounts for step-up only) |
| Purposes | `inventory.reveal`, `credentials.rotate`, `bank.change`, `withdrawal.create`, `admin.command` |
| TTL | 5 minutes (`auth.RecentProofTTL`) |
| Use | single-use consume; hash at rest in `mfa_recent_proofs` |
| Bind | user_id + session_id + purpose |
| Revoke | logout revokes open proofs for session |

Response mint metadata (once):

```json
{
  "mfaVerified": true,
  "recentMfaProof": "<opaque>",
  "purpose": "inventory.reveal",
  "expiresAt": "2026-07-17T12:44:00Z",
  "factor": "totp"
}
```

Problem codes:

- `AUTH_MFA_REQUIRED` — pending session or missing proof
- `AUTH_MFA_PROOF_INVALID` — wrong session/purpose/replay/unknown
- `AUTH_MFA_PROOF_EXPIRED` — TTL elapsed

### Consumer wiring

- Inventory reveal: `RequireRecentMFAProof(inventory.reveal)` + body `mfaVerified` **ignored** (no authority).
- Helper `middleware.RequireRecentMFAProof(purpose, validator)` ready for credentials/bank/withdrawal/admin.

### Deferred (explicit)

- Pre-enrollment ticket for invited admin without factor → **ADM-100** (login still blocks admin without MFA enrolled).
