## 3. Behavior

| Action | Transport | Post-success |
| --- | --- | --- |
| Login | `POST /v1/auth/login` body `{ email, password, surface: "ADMIN" }` | `applyLoginCsrf` + `refreshSessionAfterLogin("admin")`; redirect safe `returnTo` or `/admin` |
| login required | Login `login succeeded` | Bootstrap as authenticated session status; **no** console redirect |
| Logout | `ProfileMenu surface="admin"` → `logoutSession({ surface: "admin" })` | Server revoke + private cache clear + `/admin/login` |
| Console guard | `(console)` layout require authenticated session | Missing/wrong surface/authenticated session → `/admin/login?returnTo=…` |
| Permission UI | `AdminPermissionBoundary` | Existing panel; API fail-closed without session |

### Error disposition (no new AdminLogin regions)

| Case | Mapping |
| --- | --- |
| `AUTH_INVALID_CREDENTIALS` / 403 inactive | Result `field_errors` — stay on form (no field-error DOM yet; UXE-011) |
| `429` / unavailable / transport | `blocked` — stay on form, no fake success |
| auth pending | Stay on login; console blocked by INT-140 |
| Admin without enrolled auth | BE `403` Forbidden — maps fail; login deferred |

### Security

- Password only in form memory + request body; keys `["auth","admin","login"|"logout"]`; `gcTime: 0`
- API path: empty default credentials (mock keeps snapshot defaults for visual parity)
- Plain link bypass to `/admin` removed; CTA is submit button with same classes/copy
- “Mock access…” banner only when `auth === mock`
- `returnTo` via `sanitizeReturnToForSurface("admin")` only
- Anonymous login POST uses `skipCsrf: true`
