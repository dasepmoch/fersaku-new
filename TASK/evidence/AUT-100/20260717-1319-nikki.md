## 3. Behavior

| Action | Transport | Post-success |
| --- | --- | --- |
| Register | `POST /v1/auth/register` body `{ email, password, name, surface: "SELLER" }` | Generic success; **API** → `/login` (no session); **mock** → `/dashboard/onboarding` (prototype) |
| Login | `POST /v1/auth/login` body `{ email, password, surface: "SELLER" }` | `applyLoginCsrf` + `refreshSessionAfterLogin`; redirect safe `returnTo` or `/dashboard` |
| MFA required | Login `mfaRequired: true` | Bootstrap as `mfa_pending`; **no** dashboard redirect |
| Forgot | `POST /v1/auth/password/forgot` | Always generic success dialog when 200 |
| Logout | Existing `ProfileMenu` → `logoutSession` (INT-120) | Unchanged this task |

### Error disposition (AuthForm surfaces only)

| Case | Mapping |
| --- | --- |
| `400 VALIDATION_FAILED` | Existing field error regions |
| `AUTH_INVALID_CREDENTIALS` / unverified-style `403` | Generic message on password/email field (anti-enumeration) |
| `429` / unavailable / transport | `blocked` — stay on form, no fake success, no new panel (UXE-011) |
| MFA pending | Stay on login; dashboard blocked by INT-140 guard |

### Security

- Password only in form memory + request body; mutation keys `["auth","seller",…]` never include credentials; `gcTime: 0`
- `assertAuthMutationKeySafe` + unit tests for password leak
- `returnTo` via `sanitizeReturnToForSurface("seller")` only
- Anonymous auth POSTs use `skipCsrf: true` (BE allows unresolved cookie on login/register/forgot)
