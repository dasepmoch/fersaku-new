## 3. Behavior

| Surface | Transport | Mapping |
| --- | --- | --- |
| List | `GET /v1/buyer/purchases?limit=20` | CursorList → summary rows → `BuyerPurchase[]` (no secrets) |
| Detail (browser) | `GET /v1/buyer/purchases/{orderId}/` | Detail DTO → redacted shells; 404→null |
| Detail (SSR) | same path via `serverApiRequest` | cookie forward, `cache: no-store`; 404→`notFound()` |
| Search/filter | existing UI controls | client-side on bounded page (no new paging) |

### BoundedNoPaging launch invariant

- `PurchaseLibrary` snapshot has no `TablePagination`.
- Adapter requests `limit=20` only; does **not** follow `nextCursor`.
- Additional history requires `UI-080` or approved paging control.

### Ownership / errors

- List/detail scoped by session principal (BE buyer_user_id).
- Query keys: `["buyer", `${subjectId}:${sessionId}`, "purchases", "bounded", filters]`.
- Cross-buyer detail → `RESOURCE_NOT_FOUND` → null / `notFound()` (no enumeration).
- `401` rethrows (login flow); `403` not mapped as login.
- API errors never fall back to mock fixtures.

### Secrets

- List: no `credentialFields` / `code` / signed URLs.
- Detail base: empty credential array / empty code value / redacted download metadata.
- Raw secrets remain CHK-140 delivery access.
