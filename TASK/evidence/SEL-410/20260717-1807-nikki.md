## 1. Objective delivered

- Store-scoped withdrawal quote / create / list / lock adapters for `/dashboard/withdrawals` + `/dashboard/withdrawals/new`
- Server-authoritative integer IDR only; UI never recomputes fee/net after quote
- Quote before create; create body is `{ quoteId }` only
- Recent authentication via INT-140 `session authentication` (purpose `withdrawal.create`) — never body `reauthProof`
- UUID idempotency keys retained across timeout/retry (no `Date.now()`)
- Primary/verified bank from SEL-340 list API (not hardcoded demo bank id)
- Quote expiry gate invalidates submit; requote required
- Exact invalidate: withdrawals + finance summary + ledger + withdrawal lock
- No optimistic money mutations; success only from authoritative create result
- Mock path preserves demo fixtures + local mock create storage

## 4. Security / money rules

- No body `reauthProof` (strict BE rejects unknown fields historically)
- Create uses `requireRecentMfa: true` → header `session authentication`
- Form mints step-up with purpose `withdrawal.create` before create
- Idempotency: UUID per logical quote intent and create intent (refs survive retry)
- Mock create persists only after success; API path never optimistically mutates cache money
- Pre-quote platform fee estimate is display-only; submit uses quote server fees
