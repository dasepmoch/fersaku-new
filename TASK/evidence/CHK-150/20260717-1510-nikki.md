## 1. Objective delivered

Wire invoice surfaces so projection comes **only** from backend immutable snapshot (schema + mapper):

- `/orders/[orderId]/invoice` — session owner (order GET); guest without session **login-gated** (no guest invoice capability exchange advertised)
- `/account/purchases/[orderId]/invoice` — buyer ownership; foreign → safe `notFound()`
- `/invoices/verify/[token]` — public privacy-safe verify; invalid → existing invalid composition (never fabricate valid)

Print/download only from verified projection (`window.print`). Private SSR uses `privacy: "private"` (no-store). No UI redesign — `InvoiceView` markup/class/copy frozen; data via props only.

## 3. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Buyer invoice (api) | `GET /v1/buyer/purchases/{orderId}/invoice` | Session; foreign 404 → null/notFound |
| Order invoice (api) | `GET /v1/orders/{orderId}/invoice` | Session; 401 → buyer login + safe returnTo under `/account` |
| Public verify GET | `GET /v1/invoices/verify/{code}` | No cookies; 404 → `{ valid: false }` |
| Public verify POST | `POST /v1/public/invoices/verify` | Token in body only (adapter available) |
| Mock buyer/checkout | adapter fixtures | Deterministic demo projection; no network |
| Print | client `window.print()` | Only after valid projection loaded |

### Authority

- Gross/subtotal/tip/discount/fee from backend snapshot/header only — never sum lines client-side.
- Public verify never maps buyer email/name/secrets.
- Invalid/missing code never fabricates `valid: true`.

### Guest disposition

- No documented guest invoice capability exchange on GET invoice → **login-gate** on 401.
- Order-result CTA still links to invoice route; unauthenticated hit redirects to buyer login with return under `/account/purchases/.../invoice`.
