## 2. Tenant capability model (documented + code)

| Membership role | Store capabilities | Notes |
| --- | --- | --- |
| `OWNER` | `store.read`, `store.write` | Full ops on merchant stores |
| `STAFF` | `store.read`, `store.write` | Same store surface under current schema; no finer member-read/write until migration |
| (none) | ‚Äî | Store ID ‚Üí `RESOURCE_NOT_FOUND` |

**Precedence:** Global seller permissions (`seller.store.read` / `seller.store.write`) are surface grants only; they never authorize a foreign merchant store. Access requires active `merchant_members` for the store‚Äôs merchant. Admin bypass remains platform roles on audited paths ‚Äî not seller global perms.

Code: `backend/internal/domain/authz/store_access.go`

### Bootstrap selection (deterministic)

1. Preferred store if membership-owned 
2. Else canonical store for primary merchant 
3. Else first stable allowed store (`is_canonical DESC, created_at ASC, id ASC`)

Foreign preferred store is ignored (fallback), never leaks existence.

### Bootstrap response shape

```json
{
 "merchantId": "...",
 "displayName": "...",
 "status": "ACTIVE",
 "roleInMerchant": "OWNER",
 "ownerUserId": "...",
 "memberships": [{ "merchantId", "roleInMerchant", "capabilities", "storeIds", ... }],
 "stores": [{ "storeId", "merchantId", "slug", "name", "status", "canonical" }],
 "canonicalStoreId": "...",
 "currentStoreId": "...",
 "capabilities": ["store.read", "store.write"]
}
```
