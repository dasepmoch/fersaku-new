## 1. Objective delivered

Wire FE impersonation start/end to backend-issued derived sessions. API path treats server cookie + `/v1/auth/session` impersonation claims as authority; never trusts URL/sessionStorage alone. Mock path retains prototype local session for demos. Existing dialog/banner/policy boundary chrome unchanged.

## 3. Backend contract (consumed)

| Endpoint | Permission (BE) | Use |
| --- | --- | --- |
| `POST /v1/admin/users/{userId}/impersonation` | `impersonation.start` (+ `impersonation.support_write` for SUPPORT_WRITE) | User target start; sets derived cookie |
| `POST /v1/admin/merchants/{merchantId}/impersonation` | same | Resolve owner then start |
| `POST /v1/admin/impersonation/{sessionId}/terminate` | actor/derived holder | End + revoke derived session |
| `GET /v1/auth/session` | authenticated | Impersonation meta for banner |

Start body: `scope` READ_ONLY\|SUPPORT_WRITE, `reason` (≥12), `ticket`, `ttlMinutes` 15\|30\|60, optional `mfaCode` / `X-Recent-MFA-Proof`, idempotency.
