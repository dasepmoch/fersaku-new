## 3. Backend contract (consumed + small read-model extend)

| Endpoint | Use |
| --- | --- |
| `GET /v1/stores/{storeId}/inventory/products` | Bounded list summaries (masked counts) |
| `GET …/products/{productId}` | Summary + masked items |
| `GET/PUT …/products/{productId}/schema` | Active schema / new version (If-Match) |
| `POST …/products/{productId}/items` | Batch import (encrypted at rest) |
| `POST …/items/{itemId}/reveal` | Per-item secrets; `requireRecentMfa`; body `mfaVerified` ignored |
| `POST …/items/{itemId}/revoke` | Masked item after revoke |

**BE extend (SEL-240 checklist):** `InventoryProductSummary` now joins catalog `title` + `type` (SQL + DTO + OpenAPI). No secrets. No N+1 (single GROUP BY join).

Domain source: `sellerCatalog`.

**Reveal security:** `session authentication` via `requireRecentMfa: true` (INT-140). Secrets held only in component state with 60s TTL; never query keys/cache/storage.

**Pagination:** list has no paging control; detail stock table uses existing client `TablePagination` over masked items (BE returns bounded first page of items, limit 100).
