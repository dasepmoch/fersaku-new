## 3. Behavior

| Surface | Transport | Notes |
| --- | --- | --- |
| Poll | `GET /v1/checkout/intents/{intentId}` | Safe GET; schema + mapper; no create |
| QR | create/poll payload | `qrString` ‚Üí canvas; `qrImageUrl` ‚Üí img; else static chrome |
| Countdown | server `expiresAt` | mm:ss when present; mock keeps local timer |
| PAID | poll terminal | existing paid step + order success redirect |
| EXPIRED/FAILED/CANCELLED | poll terminal | stay on qris; no mock success |
| Poll error | backoff continue | never paid; no auto-mint intent |
| Resume | memory intent id | re-press pay reuses poll; same idempotency key until PAID |
| Mock | simulate only | never hits GET intent |

### Authority

- Only backend status `PAID` marks success.
- URL status / client timer / wallet chip cannot mark paid.
- Poll failure never creates a new intent.

### Poll policy

- Bounded exponential backoff + jitter; slower when `document.hidden`.
- Immediate refresh on `visibilitychange`‚Üívisible and `online`.
- Abort on unmount / terminal / new intent id.
- Single-flight: no overlapping GETs.
- Honors `Retry-After` when present on errors.
