## 1. Objective delivered

- Store-scoped finance summary / revenue / ledger read authority for `/dashboard/balance` + overview balance card
- Server-authoritative integer IDR only; UI never recomputes fee/net/balance from rows or source buckets
- Actor/store tenant guard on BE summary/ledger/revenue (`StoreAccessGuard` read); foreign store → safe 404
- Schema/mappers include `SETTLEMENT_RELEASE` + source `SYSTEM`→UI `MIXED`
- CursorList first-page profile for ledger (balance UI has no paging control)
- Mock path preserves demo fixtures; API path uses typed envelopes + mappers
