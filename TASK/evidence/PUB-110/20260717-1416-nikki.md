## 3. Behavior

| Surface | Source | Mapping |
| --- | --- | --- |
| Active policy | `GET /v1/platform/fees` | `feePolicyEnvelopeSchema` → DTO |
| Marketing strings | mapper | `transactionPercentBps` + `transactionFixedIdr` → `"3% + Rp700"` |
| Withdrawal | mapper | `withdrawalPercentBps` → `"3% + biaya proses"` (provider fee not guessed) |
| Minimum | mapper | `minimumWithdrawalIdr` → `rupiah(...)` → `"Rp50.000"` |
| Mock | `publicCatalog` mock domain | launch DTO (`LAUNCH_FEE_POLICY_V1`) |
| Outage (warm) | process last-known | same versioned DTO last received |
| Outage (cold) | launch fallback | release-installed `LAUNCH_FEE_POLICY_DTO` (not free-form invention) |

### Cache / revalidate

- `PUBLIC_FEE_REVALIDATE_SECONDS = 300`
- `PUBLIC_FEE_CACHE_TAG = "public-fee-policy"`
- Pricing page: `export const revalidate = PUBLIC_FEE_REVALIDATE_SECONDS`

- Marketing copy is **display only**
- Checkout quote / payment fee remains server-authoritative (CHK-100+)
- Admin fee mutation remains 405/immutable (BE); FE only GETs public fees
