5. `npm run typecheck` fails on `frontend/tests/e2e/api/qlt-220-parent-framework.spec.ts:95` — `repoPath` undefined
6. `npm run test:coverage` fails: architecture-boundary regex stale, buyer-session clock nondeterministic, int-170 regex false-positive

### Implementation steps:
1. Separate formatting source from generated/artifact directories. Add explicit ignore for `.next`, `.next-dev`, `.next-e2e-api`, reports, coverage, temp.
2. Run formatter only on files that actually change; review diff to not overwrite unrelated user changes.
3. Remove adapter import from application test — use fake/port test double. Architecture test must still enforce dependency rule.
4. Fix lint with scoped decisions: rename predicate that's not a React hook, or configure rule narrowly. Fix actual effect/render purity without disabling rule globally.
5. Fix `repoPath` typecheck and run typecheck on clean checkout after build not running parallel.
6. Replace static regex tests with parser/scoped assertions; inject clock into mapper test; tests must not depend on actual date.
7. Run backend unit/race, frontend format/lint/typecheck/coverage/build/bundle in clean environment.
