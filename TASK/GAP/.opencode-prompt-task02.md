## TASK DETAILS (from 02-P0-MALWARE-SCAN.md):

### Key findings to fix:
1. `app.go:408-419` — `LocalScanPass` only for local/test, production scanner "later"
2. `object_service.go:256-267` — always rejects completion on staging/prod with "malware scan unavailable"
3. `app.go:589-598` — `LocalScanPass` only for local/test again
4. `kyc_service.go:497-506` — production marks scan `PASSED` after heuristic EICAR/empty-shell check
5. No malware scan job in production registry (`jobs/registry.go`)

### Implementation steps:
1. Define scanner port interface: `Scan(objectRef/stream) → Verdict{CLEAN|INFECTED|ERROR|TIMEOUT}`, max bytes, timeout, retry/backoff, fail-closed policy. Scanner adapter in adapters layer, port in ports layer.
2. Scanner accepts controlled object reference/stream, NOT public URL (SSRF prevention).
3. Object quarantine until scan CLEAN. Store: scan status, verdict, engine/version, timestamp, bounded error class, checksum, operator/retry metadata. Scan failure ≠ READY.
4. KYC: do NOT encrypt then mark passed before scanner succeeds. Ensure ciphertext at-rest, minimal plaintext lifetime, zeroization/bounded memory, no payload in log/temp.
5. Worker/job retry idempotent + dead-letter/quarantine review path. Duplicate job must not change quota/grant/KYC state twice.
6. Test EICAR as infected, valid file as clean, corrupt/polyglot/oversize as rejected/error, scanner unavailable as quarantine, timeout as retry, scanner response spoof as invalid.
7. Readiness check: production not ready if scanner dependency not reachable/configured. Local fake only via explicit local/test mode.

### Acceptance criteria:
- Object upload happy path staging/prod can COMPLETE → scan CLEAN → READY; infected/timeout/error stays quarantine/rejected.
- KYC only passes after verdict CLEAN from engine/version recorded; heuristic alone does not satisfy.
- Job retry and duplicate delivery idempotent; no object READY without scan evidence.
- Metrics/alerts for queue age, scan error, infected count, quarantine backlog, latency.
- Integration test runs without vendor secret, using scanner stub simulating all verdicts.
- Runbook explains quarantine inspection, re-scan, rollback engine, incident containment.
