## TASK DETAILS (from 10-P1-SUPPLY-CHAIN.md):

### Key findings to fix:
1. `npm audit --audit-level=low` — moderate advisory on `next@16.2.10` via `postcss@8.4.31` (XSS); CI only uses `--audit-level=high`
2. Docker/compose uses mutable tags: `golang:1.25-alpine`, `alpine:3.21`, `postgres:16-alpine`, `redis:7-alpine`; image scan CI not wired
3. `backend/scripts/security_scan.sh` passes vet/govulncheck/secret pattern, but doesn't replace image CVE/SBOM/provenance scan

### Implementation steps:
1. Evaluate patched supported Next/PostCSS path; don't accept `npm audit fix` that downgrades major without compatibility review. Record advisory decision and expiry.
2. Add lockfile/package policy, `npm audit` threshold approved, OSV/SCA, review cadence. Moderate issue reachable in production must have mitigation or owner waiver.
3. Pin base/runtime images by digest; update via bot/PR that runs tests. Pin GitHub Actions by SHA per policy.
4. Generate SBOM for frontend/backend images, scan Trivy/Grype or equivalent, sign image and provenance, verify signature before deployment.
5. Scan IaC/compose/manifests for privileged mode, exposed ports, mutable tags, secret ENV, root user, debug endpoints.
6. Publish artifact evidence: dependency lock hash, SBOM digest, CVE report, base image digest, signer, approved exception expiry.

### Acceptance criteria:
- CI blocks critical/high and has explicit owner/expiry for accepted moderate advisory.
- Production release references immutable digests and signed provenance; verifier rejects unsigned artifact.
- SBOM and scan reports attached to release manifest; no secret baked in image.
- Rebuild from same source/lockfile is reproducible enough to explain digest differences.
