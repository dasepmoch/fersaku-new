## Sequence (seller create → webhook)

```mermaid
sequenceDiagram
  participant S as Seller
  participant API as WithdrawalService
  participant Fee as FeeService/ADR-0003
  participant L as Ledger
  participant X as DisbursementProvider
  participant WH as Webhook

  S->>API: CreateQuote(amount, bank, Idempotency-Key)
  API->>API: lock/frozen/emergency checks
  API->>X: QuoteDisbursement (provider fee)
  API->>Fee: CalculateWithdrawal (3% + provider; min 50_000)
  API->>L: PreviewWithdrawalAllocation + available check
  API-->>S: Quote ACTIVE (TTL default 300s)

  S->>API: RequestWithdrawal(quoteId, Idempotency-Key)
  API->>API: reject if quote expired/consumed/invalid
  API->>L: LockBalance + FIFO allocate + PostJournal RESERVE
  API->>API: status UNDER_REVIEW or APPROVED (auto)
  alt auto-approve or AdminReview approve
    API->>API: APPROVED → PROCESSING (once)
    API->>X: CreateDisbursement(external_id=fersaku_wd_{id})
    Note over API,X: timeout → UNKNOWN_OUTCOME; no second create
  end

  WH->>API: HandleDisbursementCallback(ref, status)
  alt COMPLETED
    API->>L: COMPLETE + FEE_SETTLE journals (exactly once)
    API->>API: status COMPLETED
  else FAILED
    API->>L: RELEASE reserve + restore lots (once)
    API->>API: status FAILED
  end
```

### Text sequence

1. **CreateQuote** — idempotent on key+amount+bank; fee = platform 3% (half-up bps) + provider fee; reject `< Rp50.000`; TTL from `withdrawal_quote_ttl_seconds` (default 300s).
2. **RequestWithdrawal** — consume quote once; reserve W from available via FIFO lots; status UNDER_REVIEW or APPROVED (auto).
3. **DisburseWithdrawal** — APPROVED → PROCESSING before provider call; external id stable; timeout → UNKNOWN_OUTCOME; retry does not create second payout.
4. **Webhook** — COMPLETED → complete + fee settle once; FAILED → release reserve once; replay safe.
5. **AdminReview** — HTTP `withdrawals.review` permission; reason required; approve idempotent; no MFA.

## Ledger holds/releases

| State | Reserve |
| ----- | ------- |
| REQUESTED / UNDER_REVIEW / APPROVED / HELD / PROCESSING / UNKNOWN_OUTCOME | held (`WITHDRAWAL_RESERVE`) |
| COMPLETED | complete journals; held cleared via complete legs |
| FAILED / REJECTED | `WITHDRAWAL_RELEASE` once + lot restore |
| Late COMPLETED after fail | recapture then complete |

## Composition (no fake forced when xendit)

`wireMoneyProviders` switches disbursement independently:

- `DISBURSEMENT_PROVIDER=fake` → `xendit.Fake`
- `DISBURSEMENT_PROVIDER=xendit` → `xendit.Real` (even if payment=fake|duitku)

Covered by `composition_test.go` (mixed fake pay / real disburse; duitku pay / xendit disburse). No gap forcing fake when config says xendit.

## Admin review

| Gate | Where |
