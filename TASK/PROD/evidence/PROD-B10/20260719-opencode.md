## Signature algorithms (documented)

### Create / inquiry (`POST /webapi/api/merchant/v2/inquiry`)

```text
signature = MD5( merchantCode + merchantOrderId + paymentAmount + apiKey )
```

- Concatenation is plain string (no separators).
- `paymentAmount` is the decimal integer string (e.g. `"10000"`), same value as JSON `paymentAmount` number.
- Digest is **lowercase hex** (32 chars).

## Status mapping (Duitku → domain)

| Duitku | Domain |
| ------ | ------ |
| `00` / success / paid | `PAID` |
| `01` / process / pending | `PENDING` |
| `02` + cancel message | `CANCELLED` |
| `02` / failed | `FAILED` |
| expired (message) | `EXPIRED` |
| other / empty | `UNKNOWN` |

Create success always returns domain `PENDING` (QR issued, not settled).
