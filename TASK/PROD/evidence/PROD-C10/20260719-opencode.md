## Status mapping (documented)

```
ACCEPTED|PENDING|LOCKED|REQUESTED → PENDING
PROCESSING|SENDING                 → PROCESSING
COMPLETED|SUCCEEDED|SUCCESS|PAID   → COMPLETED
FAILED|REJECTED|CANCELLED|REVERSED → FAILED
NOT_FOUND                          → NOT_FOUND
empty                              → UNKNOWN
other                              → uppercased (non-terminal apply path)
```

Apply effects (service, idempotent): COMPLETED → complete journals once; FAILED → release reserve once; PENDING/PROCESSING/UNKNOWN → bind ref / schedule lookup; no silent balance corruption.

## Fail-closed empty token

1. **Config (A20):** `validateProviderCredentials` requires `XENDIT_WEBHOOK_TOKEN` when `NeedsXenditCredentials()` (staging/production live xendit).
2. **Composition:** `effectiveWebhookToken` returns empty on staging/prod when missing (no local default).
3. **Handler (C10):** empty `WebhookToken` → 401 `token_not_configured` without calling apply. `ConstantTimeTokenEqual` rejects empty `want`.
