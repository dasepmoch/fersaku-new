## 1. Problem

- `cmd/seed` correctly **refuses** `APP_ENV=production` (exit 2).
- SUPER_ADMIN attach previously lived **only** as a post-step of seed → ops could not bootstrap prod without breaking the seed guard.
- Launch checklist still told operators to run `seed.sh` for admin bootstrap.

## 2. Solution

| Piece | Action |
|-------|--------|
| `cmd/bootstrap-admin` | **New** one-shot: `BootstrapAdminByEmail` only; no persona seed |
| Production safety | Requires `BOOTSTRAP_ADMIN_CONFIRM=yes` when `APP_ENV=production` |
| Prerequisites | User must already exist (register + verify); migrations own system roles |
| Docs | `backend/docs/launch/readiness-checklist.md` §5 rewritten |

Code review:

- `cmd/bootstrap-admin` does **not** call `seed.Apply` / personas.

## 4. Operator runbook (prod)

1. Migrate up.  
2. Register real ops email; verify.  
3. `BOOTSTRAP_ADMIN_EMAIL` + `BOOTSTRAP_ADMIN_CONFIRM=yes` + `go run ./cmd/bootstrap-admin`.  
4. Unset both env vars.  
5. Never run `cmd/seed` on production.
