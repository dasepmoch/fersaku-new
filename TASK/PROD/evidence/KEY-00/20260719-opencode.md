## 5. Environment / code — **MUST NOT** change without a **new** task

Changing any of the following during “just go live” is **scope creep**. Open a new KEY-* or PROD-* task.

| Category | Examples | Why |
|----------|----------|-----|
| Provider ports / adapters | New payment provider, Duitku payout, dual QRIS failover UI | ADR + non-goals locked |
| Money schema / ledger rules | Settlement double-entry, fee policy v2 without release process | Authority = Postgres + fee policy |
| Webhook paths | Rename `/v1/webhooks/duitku` without dashboard + client plan | Provider dashboards break |
| Auth model | Re-enable MFA as GO gate, OAuth without AUT-130 | Product: no MFA; OAuth DISABLED |
| UI freeze | Redesign checkout/dashboard/admin chrome | `TASK/00-UI-FREEZE-CONTRACT.md` |
| FE dual-mode | Ship `DATA_SOURCE=mock` or prototype stage to prod | Fail-closed live policy |
| Seed in production | Run `cmd/seed` on prod | `GuardNonProduction` |
| Deferred product surfaces | OAuth, contact submit, campaigns, avatars, playground live Send | KEY-02 / TASK/10 |
| Live money without GO | Agent-run KEY-62 / G40 | Human `GO LIVE CANARY` only |
| Secrets in git/evidence | Paste keys into MD, Docker image, logs | Hard ban |
