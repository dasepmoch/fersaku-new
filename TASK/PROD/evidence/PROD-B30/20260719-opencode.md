## Summary

Checkout and gateway intent creation no longer hardcode `Provider: XENDIT`. Both services take `PaymentProvider` + `PaymentAccountScope` from composition, driven by `PAYMENT_PROVIDER`. When `PAYMENT_PROVIDER=duitku`, intents persist `provider=DUITKU` and `account_scope=duitku-primary` (or `DUITKU_ACCOUNT_SCOPE`) so B20 callbacks resolve via the provider/scope tuple.

QRIS adapter injection was already correct (B10 `wireMoneyProviders`); this task only fixed **intent identity** on create.

## Provider selection

```text
PAYMENT_PROVIDER=fake    → ProviderXendit + xendit-primary (xendit.Fake)
PAYMENT_PROVIDER=xendit  → ProviderXendit + XenditAccountScope
PAYMENT_PROVIDER=duitku  → ProviderDuitku + DuitkuAccountScope (default duitku-primary)
```

Composition helper: `paymentIntentIdentity(cfg, xenditAccountScope)` in `app.go`.

Services use:

```go
Provider:     coalesce(PaymentProvider, ProviderXendit)
AccountScope: coalesce(PaymentAccountScope, AccountScope, xendit-primary)
```

`CreateQRIS` input `AccountScope` matches the same payment scope.
