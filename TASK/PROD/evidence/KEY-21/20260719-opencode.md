## 2. Generate (ops — do not paste into git)

Prefer high-entropy secrets (≥32 random bytes, hex or base64). Any non-empty string works (SHA-256 derived), but **do not** use short/placeholder strings.

```bash
# recommended: 32-byte hex (64 chars) or base64
openssl rand -hex 32
# or
openssl rand -base64 32
```

Generate **two independent** secrets:

| Env | Purpose |
|-----|---------|
| `KYC_ENCRYPTION_KEY` | KYC + fallback for stock |
| `STOCK_ENCRYPTION_KEY` | Inventory CODE envelopes (recommended separate) |

Store **only** in secret manager (KEY-10 paths). Never commit. Never put in evidence.

## 4. Rotation policy

| Event | Action |
|-------|--------|
| Leak suspicion | Generate new keys → dual-read period **requires engineering re-encrypt plan** (not flip-only) |
| Routine | Prefer rare; ciphertext is bound to key material via SHA-256 |
| KEY-62 live canary | Use **same** staging-proven key material or documented re-encrypt; do not casually rotate mid-canary |

**Important:** Changing KYC/STOCK keys without re-encrypt makes old ciphertext unreadable. Rotation ≠ key-swap of Duitku/Xendit.
