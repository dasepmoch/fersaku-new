## Limiter design review

| Surface | Implementation |
| ------- | -------------- |
| Global HTTP (login/checkout/etc.) | `middleware.RateLimit` on client IP (`ratelimit.go`) |
| Local/test default | in-memory token bucket **capacity 120, refill 20/s** (`router.go` / `app.go`) |
| Staging/production | Redis token bucket **cap 120 / window 1m** via `RequiresDistributedLimiter()`; **fail-closed** if Redis missing |
| Gateway merchant | in-service `GatewayService.checkRateLimit` (separate map) |
| Webhooks | same IP middleware (no separate unlimited bypass in router stack) |

## Retry-After behavior

On deny:

1. HTTP **429**
2. Problem code **`RATE_LIMITED`**
3. Header **`Retry-After`** = seconds until ≥1 token (minimum 1)

Documented in `middleware/ratelimit.go` L76–100. Asserted by `TestRateLimit` (`router_smoke_test.go`).

```bash
go test ./internal/adapters/http/ -count=1 -run TestRateLimit   # PASS
go test ./internal/app/ -count=1 -run TestRateLimiter           # PASS (local + Redis fail-closed)
```
