## Duitku callback auth (document in evidence)

Common Duitku callback fields (form or JSON depending on product — support **both** JSON and `application/x-www-form-urlencoded`):

Typical fields:
- `merchantCode`
- `amount` / `paymentAmount`
- `merchantOrderId`
- `productDetail` / `productDetails`
- `additionalParam`
- `paymentCode` / `paymentMethod`
- `resultCode` (`00` success, `01` pending, `02` failed/canceled)
- `merchantUserId`
- `reference`
- `signature`
- `publisherOrderId` (sometimes)
- `spUserHash` (sometimes)
- `settlementDate`
- `issuerCode`

**Callback signature (common Duitku v2):**

```text
MD5( merchantCode + amount + merchantOrderId + apiKey )
```

- `amount` as string without decimals for whole IDR (same as posted amount field)
- lowercase hex
- constant-time compare of computed vs provided signature

Also validate `merchantCode` matches configured `DUITKU_MERCHANT_CODE` (constant-time or subtle.ConstantTimeCompare on equal-length padded carefully — if lengths differ, reject).

Export helper from `adapters/duitku` e.g. `CallbackSignature(merchantCode, amount, merchantOrderId, apiKey string) string` and `VerifyCallbackSignature(...) bool`.

If amount field name varies, normalize from `amount` or `paymentAmount`.
